Bookshelf

Aug 22nd, 2014 / angular, express, fullstackJS, mongodb, node, project, webapp

Worked through the yeoman tutorial. In my opinion not useful until understanding of the mean stack is already mastered – took about 1.5 hours. Worked through two tutorials on scotch.io, starting to get how the parts of the application are put together – the node API queries the MongoDB database and provides CRUD operations for the Angular part to use.

Will need to figure out a way to query an open database for books when users type in book title, author, etc. will need functionality in the get function so users can get by title, author and others. First, work through the API creation tutorials and user auth as these are critical parts of any application.

Idea is to work through enough tutorials to understand how to create a basic CRUD app, and then start to focus specifically on bookshelf. Goal for tomorrow (Saturday, August 23rd) is to finish the back end tutorials, including user auth, set up the Schemas for bookshelf and then start working through the Angular tutorial. Ideal case is to actually finish the Angular tutorial.

Finished the Node API tutorial. I get how to create an api now that can CRUD. For bookshelf, each new user will have to have a json object containing an array of book objects which point to a book. Need to figure out how to create this complex structure. Not sure how to actually use the api in the front-end of the application yet.

It is 08-26, 4 days since I started this project and the API is nearly complete. It currently supports requesting and adding friends, adding (and automatically creating if DNE) books to lists, deleting books from lists and creating, reading and deleting user information. Final components of API are to allow users to delete friends and to update their email. Current user model identifies users based on username which is convenient since they must be unique. Going to start version controlling this app now that I’ve grasped the basics of the back end.
